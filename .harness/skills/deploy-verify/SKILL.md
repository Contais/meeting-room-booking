# 部署验证技能 (Deployment Verification)

## 概述
在代码合并至主分支或发布上线前，执行一套标准化的冒烟测试（Smoke Test）和环境健康检查，确保应用可用性。

## 触发条件
当收到部署流水线（CI/CD Pipeline）触发通知时。

## 执行步骤

### Step 1: 环境健康检查 (Health Check)
- **端口监听**：验证服务是否正常绑定端口。
- **数据库连接池**：验证数据源配置是否正确，能否成功获取连接。
- **Redis 连接**：验证 Lettuce/Jedis 客户端是否能正常 ping 通 Redis。
- **注册中心**：验证服务是否成功注册到 Nacos/Eureka。

### Step 2: API 冒烟测试 (Smoke Test)
- **根路径测试**：GET /actuator/health 或 API 文档首页是否返回 200。
- **核心链路测试**：
  - 用户登录接口：验证 JWT Token 生成是否正常。
  - 核心业务接口（如商品列表、下单）：验证接口响应时间在 500ms 以内，返回数据结构符合预期。
- **错误码测试**：验证传入错误参数时，全局异常处理器是否能正确拦截并返回标准错误码。

### Step 3: 基础设施依赖验证
- **分布式锁测试**：在高并发场景下，验证 Redisson 锁是否能正常生效，无死锁现象。
- **消息队列测试**：验证 RocketMQ Producer 发送消息无阻塞，Consumer 消费无积压。
- **定时任务测试**：验证 @Scheduled 任务是否正常触发，无重复执行或漏执行。

### Step 4: 监控告警配置检查
- **日志格式验证**：检查日志是否输出了 JSON 格式，且包含 traceId。
- **Metrics 暴露**：检查 Actuator 端点是否暴露了 Prometheus 指标。
- **告警规则**：确认 CPU > 80%、内存 > 90%、错误率 > 1% 等告警规则已生效。
