# 专家评审技能 (Expert Reviewer)

## 概述
针对核心业务逻辑、复杂算法和架构重构方案，提供深度的技术评审意见，确保技术方案的先进性与可行性。

## 触发条件
当涉及到以下场景时触发：
- 核心领域模型（Domain Model）发生重大变更。
- 引入新的中间件（如 Elasticsearch, MongoDB）。
- 涉及金额计算的核心算法重构。
- 微服务拆分或合并方案评审。

## 评审重点

### 1. 领域驱动设计 (DDD) 符合度
- **聚合根设计**：检查实体之间的关联关系是否合理，是否避免了循环依赖。
- **防腐层 (ACL)**：检查与外部系统的交互是否通过明确的 Adapter/Client 封装，避免业务逻辑中混杂 HTTP/RPC 调用代码。

### 2. 并发与一致性
- **分布式事务**：评估强一致性（Seata）与最终一致性（MQ）的选择是否合理。
- **并发控制**：检查多线程场景下的线程安全问题（如使用 ConcurrentHashMap 还是 Collections.synchronizedMap）。

### 3. 扩展性考量
- **设计模式应用**：检查是否预留了扩展点（如使用策略模式处理不同的促销折扣计算）。
- **插件化架构**：评估核心流程是否支持热插拔，是否过度耦合。

### 4. 技术债评估
- **复杂度评估**：使用圈复杂度（Cyclomatic Complexity）工具检查代码逻辑是否过于复杂，是否需要重构。
- **依赖分析**：检查是否存在循环依赖或过度依赖底层基础架构。

## 输出
生成《专家评审意见书》，包含：
- 技术方案的优缺点分析
- 潜在风险预警及规避建议
- 代码重构的具体实施路径
