# 领域术语 (Domain Glossary)

## 1. 核心业务术语

| 术语 | 英文/缩写 | 定义 | 代码体现 |
|------|-----------|------|----------|
| 会议室 | Meeting Room | 用于会议的房间，包含座位、投影仪、白板等设施。 | `MeetingRoom` 实体 |
| 预约 | Reservation | 用户预约会议室的记录。 | `Reservation` 实体 |
| 幂等性 | Idempotency | 同一操作执行一次与执行多次的结果相同。 | MQ Consumer 必须实现幂等 |

## 2. 状态定义

### 2.1 会议室状态 (Meeting Room Status)
- **DISABLED (0)**: 禁用。会议室已被禁用，用户无法预约。
- **ENABLED (1)**: 启用。会议室已被启用，用户可预约。

### 2.2 会议室预约状态 (Reservation Status)
- **PENDING_CONFIRMATION (0)**: 待确认。预约已创建，等待管理员确认。
- **CONFIRMED (1)**: 已确认。管理员已确认预约，用户可使用。
- **CANCELLED (2)**: 已取消。用户主动取消预约或预约超时未确认。

## 3. 技术术语

| 术语 | 解释 | 代码/配置示例 |
|------|------|----------------|
| DTO | Data Transfer Object，数据传输对象。 | `MeetingRoomDTO`, `ReservationDTO` |
| DO/Entity | Domain Object，领域实体。 | `MeetingRoom`, `Reservation` |
| VO | View Object，视图对象。 | `MeetingRoomVO`, `ReservationVO` |
| 聚合根 | Aggregate Root，领域模型的入口点。 | `Order` 是 `OrderItem` 的聚合根。 |
| 值对象 | Value Object，无唯一标识的对象。 | `Money` (金额), `Address` (地址) |

## 4. 命名约定 (Naming Conventions)
1. **金额字段**: 必须以 `{attribute}Cents` 结尾 (如 `priceCents`, `amountCents`)。
2. **布尔值**: 使用 `is` 或 `has` 前缀 (如 `isDeleted`, `hasStock`)。
3. **时间字段**: 使用 `Time` 后缀 (如 `createTime`, `payTime`)。
4. **ID**: 使用 `Id` 后缀 (如 `userId`, `orderId`)。
5. **命令/事件**: 使用动词过去式或将来式 (如 `CreateOrderCmd`, `OrderPaidEvent`)。
