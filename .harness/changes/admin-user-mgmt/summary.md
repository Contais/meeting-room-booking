# 管理端用户管理 - 需求分析报告

## 1. 需求描述

为管理员提供用户管理功能，支持对系统用户的全生命周期管理，包括查看、新增、编辑、启用/禁用、删除。

### 验收标准

| 编号 | 验收条件 |
|------|----------|
| AC-1 | 管理员可分页查询用户列表，支持按用户名/手机号搜索、按状态筛选 |
| AC-2 | 管理员可新增用户，指定用户名、密码、手机号、角色 |
| AC-3 | 管理员可编辑用户信息（手机号、角色） |
| AC-4 | 管理员可启用/禁用用户，禁用后该用户无法登录 |
| AC-5 | 管理员可逻辑删除用户，删除前检查是否存在未完成预约 |
| AC-6 | 管理员可查看用户详情 |
| AC-7 | 非 admin 角色调用管理接口返回 403 |
| AC-8 | JWT Token 中携带 role 信息 |
| AC-9 | 前端根据角色动态渲染菜单（admin 看到用户管理入口） |

## 2. 技术变更清单

### 2.1 用户实体变更

| 变更项 | 说明 |
|--------|------|
| User.role 字段 | 新增 VARCHAR(20)，默认 'user'，枚举值: admin/user |
| User.realName 真实姓名 | 新增 VARCHAR(64)，可选 |
| init.sql | 新增 admin 管理员账号，role='admin' |

### 2.2 后端接口

**用户管理接口（需 admin 角色）：**

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/user/admin/list | GET | 分页查询用户列表 |
| /api/user/admin/detail/{id} | GET | 查询用户详情 |
| /api/user/admin/create | POST | 新增用户 |
| /api/user/admin/update | PUT | 编辑用户 |
| /api/user/admin/toggle-status/{id} | PUT | 启用/禁用用户 |
| /api/user/admin/delete/{id} | DELETE | 逻辑删除用户 |

**鉴权变更：**

| 变更项 | 说明 |
|--------|------|
| JWT 签发 | AuthServiceImpl.generateToken() 携带 role |
| JwtUtils | 新增 getRoleFromToken() |
| 角色校验 | 新增注解 @RequiresRole 或拦截器 |

### 2.3 前端变更

| 变更项 | 说明 |
|--------|------|
| 用户管理页面 | 用户列表 + 新增/编辑弹窗 |
| 路由守卫 | admin 角色才能访问 /admin/* 路径 |
| 菜单动态渲染 | 根据 role 显示/隐藏管理菜单 |
| 类型定义 | User 类型增加 role 字段 |

## 4. 冲突与风险

### 4.1 逻辑删除流程

**删除用户时的关联数据处理：**
- 用户的历史预约记录：**保留**（预约记录中的 userId 指向已删除用户，前端显示"已注销用户"）
- 用户的进行中预约：**拒绝删除**，需先取消或完成预约
- Redis 中的 Token：**同步清除**

**删除流程：**
```
1. 查询用户是否有 status=0(待确认) 或 status=1(已确认) 的预约
2. 如有 → 返回错误 "该用户有未完成的预约，请先处理"
3. 如无 → 标记 deleted=1，清除 Redis Token
```

### 4.2 安全风险

| 风险 | 缓解措施 |
|------|----------|
| 管理接口未校验角色 | 网关/拦截器统一校验 JWT 中的 role |
| 管理员无法被删除 | 删除逻辑中判断目标用户 role=ADMIN 时拒绝 |
| 密码明文传输 | 注册/创建接口密码字段应加密或走 HTTPS |
